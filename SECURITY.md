# Security Policy

This repository contains educational material and offline examples. Do not put real credentials, customer data, proprietary prompts, or production endpoints into issues, pull requests, or challenge fixtures.

If you find a vulnerability in executable code, please report it privately to the maintainers rather than opening a public issue. Include the affected path, reproduction steps, impact, and a suggested mitigation when possible. Do not test against systems you do not own.

The examples intentionally treat tool input as untrusted. Before adapting them to production, add authentication, authorization, rate limits, secret isolation, audit logging, and provider-specific security controls.
