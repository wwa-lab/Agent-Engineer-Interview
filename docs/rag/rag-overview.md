# RAG Overview

RAG combines ingestion, indexing, retrieval, context assembly, generation, and evidence presentation. Quality can fail at every boundary: stale documents, bad chunks, missing permissions, poor ranking, context distraction, or unsupported synthesis. Measure each stage rather than blaming the model.

Start with a small representative dataset and include no-answer and conflicting-evidence cases. See [chunking](chunking.md), [retrieval](retrieval.md), and [failure modes](rag-failure-modes.md).
