# Coding Interview Prompts

These prompts are intentionally small enough to implement in 30–45 minutes. State invariants and test failure paths.

## Interview Question

Implement a bounded tool-calling loop that validates arguments and stops on a budget.

## Difficulty

Beginner to Senior

## Type

Coding

## Follow-up Questions

How would you make writes idempotent? How would you add authorization? Which parts are deterministic?

### Prompts

1. Parse a typed tool request and return a clear validation error.
2. Implement cosine similarity with zero-vector handling.
3. Add a memory store with session scope and a bounded long-term policy.
4. Build a JSON-RPC dispatcher that rejects unknown methods.
5. Compare expected and actual agent outcomes and report failures by category.
